# Phase 7: Verification, Deb Bump & Docs

## Overview

End-to-end verification on GNOME 49/50 Wayland, version bump, rebuild `.deb`, update
README/docs so install and features match v1.1.

## Requirements

- Functional: all phase success criteria re-checked on a clean nested shell and/or live
  session after `dev-install.sh`.
- Packaging: deb version `1.1-1` (or `1.0-2` if preferred — **use 1.1-1**); extension

## Implementation Steps

1. Freeze features; `rg` for leftover `log(\`[Clipboard]` direct calls; debug flag respected.
2. Manual QA checklist (below).
3. Bump versions + debian changelog.
4. `build-deb.sh`; install deb on test machine or verify package contents
   (`dpkg-deb -c`).
5. README: document GSettings keys, shortcuts, pause, image pins location, debug.
6. Final `git status` hygiene (no build artifacts committed unless project already tracks
   `dist/` — currently tracks `dist/clipboard-extension_1.0-1_all.deb`; replace/update
   consistently with project habit).
